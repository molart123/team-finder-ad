# 🔍 TeamFinder

**TeamFinder** — веб-приложение для поиска команд и тиммейтов. Пользователи могут создавать объявления о поиске
участников в команду или искать уже существующие команды по своим интересам и навыкам.

---

## 📋 Функциональность

### Проекты
- Лента актуальных открытых проектов с пагинацией (12 на страницу), сортировка от новых к старым
- Страница проекта: название, описание, автор, дата публикации, статус (открыт / закрыт)
- Создание и редактирование проекта (только для авторизованных пользователей)
- Завершение проекта владельцем (смена статуса на «закрыт»)
- Кнопка «Присоединиться к проекту» — участие без перезагрузки страницы
- Список участников проекта с аватарами и ссылками на профили
- Ссылка на GitHub-репозиторий проекта
- Иконка «Поделиться ссылкой» на проект (копирование в буфер обмена)

### Избранное
- Добавление / удаление проекта в избранное с главной страницы и со страницы проекта
- Страница «Избранное» — список сохранённых проектов (доступна только владельцу)
- Незалогиненный пользователь перенаправляется на страницу входа при попытке добавить в избранное

### Пользователи
- Список всех зарегистрированных пользователей с пагинацией (12 на страницу)
- Фильтрация пользователей по четырём критериям (только для авторизованных):
    - «Авторы избранных проектов»
    - «Авторы проектов, в которых я участвую»
    - «Пользователи, которым нравятся мои проекты»
    - «Участники моих проектов»
- Публичный профиль пользователя: имя, фамилия, аватар, о себе, контакты (email, телефон, GitHub), список созданных
  проектов
- Иконка «Поделиться профилем» (копирование ссылки в буфер обмена)

### Аккаунт
- Регистрация по email, имени и фамилии
- Вход и выход из системы
- Редактирование профиля: имя, фамилия, аватар, о себе, телефон, GitHub
- Смена пароля
- Автоматическая генерация аватара с инициалом при регистрации

### Разграничение прав
- **Гость:** просмотр проектов, страниц пользователей, регистрация
- **Авторизованный пользователь:** создание и управление своими проектами, участие в чужих, избранное
- **Администратор:** полный доступ через Django Admin Panel

## 🚀 Развёртывание

### 1. Клонировать репозиторий

```bash
git clone https://github.com/molart123/team-finder-ad.git
cd team-finder-ad
```

### 2. Виртуальное окружение

Создайте и активируйте виртуальное окружение:

```bash
python3 -m venv venv
```

- **Windows (PowerShell):** `venv\Scripts\Activate.ps1`
- **Windows (cmd):** `venv\Scripts\activate`
- **Linux/Mac:** `source venv/bin/activate`

Установите зависимости:

```bash
pip install -r requirements.txt
```

### 3. Настройка `.env`

Скопируйте пример и заполните своими значениями:

```bash
cp .env_example .env
```

| Переменная             | Назначение                                                                                                 |
|------------------------|------------------------------------------------------------------------------------------------------------|
| `DJANGO_SECRET_KEY`    | Секретный ключ Django. Можно сгенерировать через `get_random_secret_key` из `django.core.management.utils` |
| `DJANGO_DEBUG`         | Режим отладки. Установите `True` во время разработки.                                                      |
| `POSTGRES_DB`          | Имя базы данных PostgreSQL.                                                                                |
| `POSTGRES_USER`        | Имя пользователя PostgreSQL.                                                                               |
| `POSTGRES_PASSWORD`    | Пароль пользователя PostgreSQL.                                                                            |
| `POSTGRES_HOST`        | Адрес сервера БД. При локальной разработке — `localhost`.                                                  |
| `POSTGRES_PORT`        | Порт подключения к БД (по умолчанию `5432`).                                                               |
| `DJANGO_ALLOWED_HOSTS` | Список допустимых хостов через запятую (например, `localhost,127.0.0.1`).                                  |

### 4. Запуск PostgreSQL через Docker

```bash
docker compose up -d
```

База данных будет доступна по адресу `localhost:5432`.

Для остановки:

```bash
docker compose down
```

> **Примечание:** если порт 5432 уже занят, измените маппинг в `docker-compose.yml` на нестандартный (например,
`"5433:5432"`) и укажите тот же порт в `.env`.

> **Ошибка "permission denied"?** Добавьте `sudo` перед командой docker.

### 5. Применить миграции и запустить сервер

```bash
python manage.py migrate
python manage.py runserver
```

Приложение будет доступно по адресу: [http://localhost:8000](http://localhost:8000)
